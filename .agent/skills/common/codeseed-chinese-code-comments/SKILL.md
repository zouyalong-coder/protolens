# Codeseed Chinese Code Comments

当任务会产生或修改任何代码、脚本、配置代码或示例代码时，使用这个 skill。

## 全局要求

1. 新增代码文件的顶部必须添加中文文件说明，概括文件职责、主要入口或使用场景。
2. 修改已有代码文件时，如果文件已有头部说明，保持现有风格并在需要时补充中文说明；如果新增的代码片段形成新的主要职责，应补充中文文件说明。
3. 关键业务逻辑、边界条件、错误处理、状态转换、并发/异步流程和外部系统交互必须添加中文注释。
4. 关键结构体、类、枚举、类型别名、接口、trait、schema 和公开数据模型必须添加中文注释，说明用途和重要字段语义。
5. 注释解释“为什么”和“关键约束”，避免重复描述显而易见的语法。
6. 用户或项目规范明确要求其他语言注释时，优先遵守更具体的项目规范；否则默认使用中文注释。

## 输出习惯

1. 提交代码前检查新增文件是否有中文文件头说明。
2. 检查新增关键结构体和关键逻辑是否有中文注释。
3. 对小型、显而易见的局部变量或简单 getter/setter，不强行添加噪音注释。
4. 生成示例代码或文档代码块时，同样遵守中文注释要求。
