# ProtoLens 产品简报

ProtoLens 是一个跨平台抓包与流量检查工具，重点是易用、可扩展、以本地工作流为核心。第一阶段产品形态是命令行工具，但核心能力必须能被未来的桌面程序或本地服务复用。

## 目标

- 提供简单好用的 CLI，用于捕获和检查本机与局域网流量。
- 支持局域网抓包场景，包括通过代理或网关方式抓手机流量。
- 将抓包、协议解析、TLS/HTTPS 解密、分析、存储、展示拆成独立模块。
- 协议支持采用插件化方向，新增协议分析器时不改动抓包主流程。
- 为后续 AI 分析、协议洞察、桌面 UI 集成预留稳定接口。
- 为未来 TUN 模式预留扩展点。

## 第一版不做

- 不追求 Wireshark 级别的完整协议覆盖。
- 不做企业级代理管理平台。
- 不做所有平台上的透明全局拦截。
- 不做 GUI 或桌面程序打包。
- 不开发内核驱动。
- 不做 QUIC/HTTP3 解密。

## 第一版范围

- Rust CLI 应用。
- TCP 作为第一个支持的传输层目标。
- 基础 packet/session 模型，用于聚合 TCP flow。
- 在权限允许的情况下从指定网卡抓包。
- 手机抓包功能可以后置；实现时优先采用显式代理路径，而不是透明局域网抓包。
- 先完成 HTTPS 拦截方案设计；实现可以放在基础 TCP 抓包之后。
- 将捕获事件导出为稳定结构化格式，例如 JSON Lines，方便未来 UI 和分析复用；payload 可以保留，但必须能表达非 UTF-8 或不可读二进制内容。

## 未来范围

- TLS MITM 代理，本地 CA 生成，以及证书安装指引。
- HTTP/1.1、HTTP/2、WebSocket、DNS、TLS metadata 和自定义协议分析器。
- TUN 模式，用于更接近系统级的流量接入。
- 复用核心引擎的桌面程序。
- 第三方分析器插件运行时；第一阶段只做静态 Rust 插件。
- 面向会话的 AI 分析，包括脱敏摘要、异常检测、协议解释。

## 产品原则

- 首次使用要简单：CLI 应引导用户处理权限、网卡选择和常见抓包模式。
- 内部模块化：主要能力都应能脱离 CLI 单独复用。
- 明确安全边界：HTTPS 拦截必须显式开启，并清楚说明根证书信任影响。
- 可移植核心，平台差异隔离：OS 相关的抓包、代理、证书、TUN 逻辑放在 trait 或适配层后面。
- 稳定事件模型：一次捕获，多次分析。

## 典型场景

- 抓本机流量用于调试。
- 在电脑上启动局域网代理，让手机配置代理后检查移动 App 的 TCP/HTTPS 流量。
- 保存会话用于离线分析。
- 后续添加协议分析器，而不改动抓包引擎。
- 将选定会话输入 AI 分析模块。
